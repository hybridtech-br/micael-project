# Contribuindo com o HYBRID Standards

## Fluxo de mudança

1. Abra uma RFC para alterações normativas relevantes.
2. Registre um ADR quando houver decisão arquitetural.
3. Crie uma branch curta e focada.
4. Atualize documentação, exemplos e changelog.
5. Abra uma Pull Request com impacto, riscos e validações.

## Regras

- não incluir regras de negócio específicas de produtos;
- manter linguagem clara e verificável;
- preservar compatibilidade ou documentar a quebra;
- referenciar issues, RFCs e ADRs relacionados;
- validar links e Markdown antes do merge.

## Revisão

Mudanças normativas exigem revisão arquitetural. Correções editoriais simples podem seguir revisão documental comum.
